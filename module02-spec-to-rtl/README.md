## Outcomes

After this module you can explain and practice the ideas taught by **`spec-to-rtl`**, in the browser and/or with a UART RTL/TB sketch.

## Two tracks (pick one or both)

### Track A — Real RTL/TB (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and work the prompts (paper + optional `.v`).
2. Complete [CHECKLIST.md](CHECKLIST.md); use  (archive; prefer **learn_uart** / **learn_spi** / **learn_i2c**) when helpful.
3. Optional self-check: `./scripts/module.sh 02 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/spec-to-rtl/index.html](http://127.0.0.1:8080/tools/spec-to-rtl/index.html)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/spec-to-rtl/](https://universal-verification-methodology.github.io/learning/tools/spec-to-rtl/)
3. Load the **starter example**, then work challenges (or note Coming soon).
4. Check off the Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs are literacy tools — they do not replace a synthesizable UART + self-checking TB.
