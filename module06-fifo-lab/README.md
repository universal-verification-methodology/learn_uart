## Outcomes

After this module you can explain and practice the ideas taught by **`fifo-lab`**, in the browser and/or with a UART RTL/TB sketch.

## Two tracks (pick one or both)

### Track A — Real RTL/TB (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and work the prompts (paper + optional `.v`).
2. Complete [CHECKLIST.md](CHECKLIST.md); use [`../learn_uart_spi_i2c/`](../learn_uart_spi_i2c/) when helpful.
3. Optional self-check: `./scripts/module.sh 06 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/fifo-lab/index.html](http://127.0.0.1:8080/tools/fifo-lab/index.html)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/fifo-lab/](https://universal-verification-methodology.github.io/learning/tools/fifo-lab/)
3. Load the **starter example**, then work challenges (or note Coming soon).
4. Check off the Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs are literacy tools — they do not replace a synthesizable UART + self-checking TB.
